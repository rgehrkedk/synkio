# Synkio v2 Examples

This document provides practical examples of how to integrate and use the design tokens generated by Synkio v2 in your TypeScript projects.

## Use Case 1: Basic Project Setup

This is the most common setup for a standalone web application (e.g., a Next.js or Vite app).

### `tokensrc.json`

First, configure Synkio to output tokens into a directory your application can access.

```json
{
  "version": "2.0.0",
  "figma": {
    "fileId": "your-figma-file-id",
    "nodeId": "your-figma-node-id",
    "accessToken": "${FIGMA_TOKEN}"
  },
  "output": {
    "dir": "src/styles/tokens",
    "format": "json"
  }
}
```

### `tsconfig.json`

To allow TypeScript to import JSON files directly, set `resolveJsonModule` to `true`.

```json
{
  "compilerOptions": {
    "target": "es5",
    "lib": ["dom", "dom.iterable", "esnext"],
    "allowJs": true,
    "skipLibCheck": true,
    "strict": true,
    "forceConsistentCasingInFileNames": true,
    "noEmit": true,
    "esModuleInterop": true,
    "module": "esnext",
    "moduleResolution": "node",
    "resolveJsonModule": true, // <-- Important!
    "isolatedModules": true,
    "jsx": "preserve",
    "incremental": true
  },
  "include": ["next-env.d.ts", "**/*.ts", "**/*.tsx"],
  "exclude": ["node_modules"]
}
```

### Usage in Code

Now you can import the tokens directly into your components or styling logic.

```typescript
// src/components/Button.tsx
import React from 'react';
import colors from '../styles/tokens/colors.json';
import spacing from '../styles/tokens/spacing.json';

const Button = ({ children }) => {
  const style = {
    backgroundColor: colors.primary.main.value,
    color: colors.neutral.white.value,
    padding: `${spacing.sm.value}px ${spacing.md.value}px`,
    border: 'none',
    borderRadius: '4px',
  };

  return <button style={style}>{children}</button>;
};

export default Button;
```

## Use Case 2: Using Path Aliases

For cleaner imports, you can configure path aliases in your `tsconfig.json`.

### `tsconfig.json` with `baseUrl` and `paths`

```json
{
  "compilerOptions": {
    // ... other options
    "resolveJsonModule": true,
    "baseUrl": ".", // <-- Set the base directory
    "paths": {
      "@/tokens/*": ["src/styles/tokens/*"] // <-- Define the alias
    }
  },
  "include": ["next-env.d.ts", "**/*.ts", "**/*.tsx"],
  "exclude": ["node_modules"]
}
```

### Usage with Aliases

This makes your import statements much cleaner and less dependent on relative paths.

```typescript
// src/components/Button.tsx
import React from 'react';
import colors from '@/tokens/colors.json'; // <-- Cleaner import
import spacing from '@/tokens/spacing.json'; // <-- Cleaner import

const Button = ({ children }) => {
  // ... same component code
};

export default Button;
```

## Use Case 3: Monorepo Setup

In a monorepo, you can have a dedicated package for your design tokens that other packages in the monorepo can consume.

### Directory Structure

```
/
├── packages/
│   ├── design-system/
│   │   ├── src/
│   │   │   └── tokens/ (output from Synkio)
│   │   ├── package.json
│   │   └── tsconfig.json
│   └── web-app/
│       ├── src/
│       ├── package.json
│       └── tsconfig.json
├── tokensrc.json (at the root)
└── package.json
```

### Root `tokensrc.json`

Configure Synkio to output tokens into the `design-system` package.

```json
{
  "version": "2.0.0",
  "figma": {
    // ... figma config
  },
  "output": {
    "dir": "packages/design-system/src/tokens",
    "format": "json"
  }
}
```

### `packages/design-system/tsconfig.json`

The `tsconfig.json` in your `design-system` package should be configured to build the token files.

```json
{
  "compilerOptions": {
    "outDir": "dist",
    "rootDir": "src",
    "resolveJsonModule": true,
    "esModuleInterop": true
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "dist"]
}
```

### `packages/web-app/tsconfig.json`

The `web-app` can then reference the `design-system` package.

```json
{
  "compilerOptions": {
    // ... other options
    "baseUrl": ".",
    "paths": {
      "@/design-system/*": ["../design-system/src/*"]
    }
  },
  "references": [
    { "path": "../design-system" } // <-- TypeScript project reference
  ]
}
```

### Usage in the Web App

After setting up project references, you can import tokens from the shared package.

```typescript
// packages/web-app/src/components/Header.tsx
import React from 'react';
import colors from '@/design-system/tokens/colors.json';

const Header = () => {
  return <header style={{ backgroundColor: colors.primary.main.value }} />;
};
```
