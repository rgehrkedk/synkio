# Onboarding Redesign Requirements

## Overview

Replace the current wizard-based onboarding (Welcome â†’ Select â†’ Guide) with a **tab-based approach** that uses the real app UI. First-time users are guided through the tabs with contextual prompts, but the UI remains the same as post-onboarding.

## Core Concept

- **No separate onboarding screens** - just the app with first-time guidance
- **Three permanent tabs**: Sync, Tokens, Setup
- **First-time flow**: Guide users through tabs sequentially (1 of 3 â†’ 2 of 3 â†’ 3 of 3)
- **After onboarding**: Same tabs, no guidance overlay

| Tab | First-Time Additions | Removed After Onboarding |
|-----|---------------------|--------------------------|
| **Sync** | Welcome card, Website/Docs links, "1 of 3", "NEXT" CTA | Welcome card, links, badge |
| **Tokens** | "2 of 3", "NEXT" CTA | Badge only |
| **Setup** | "(optional)" text, skip subtext, "3 of 3", "START SYNCING" CTA | Optional text, subtext, badge |

The core UI (tabs, cards, forms, checkboxes) is **identical** - only contextual text and CTAs change.

## Implementation Notes

### Code Structure (No Redundancy)

Each tab screen should handle both first-time and normal states internally:

```typescript
// Example: SyncScreen
function SyncScreen(state: PluginState, actions: RouterActions): HTMLElement {
  const { isFirstTime, onboardingStep } = state;

  // Content area - same for both states
  const content = buildSyncContent(state);

  // Footer changes based on state
  const footer = isFirstTime && onboardingStep === 1
    ? buildFirstTimeFooter('NEXT: SELECT TOKENS', () => {
        actions.navigate('tokens');
        actions.updateState({ onboardingStep: 2 });
      })
    : buildNormalFooter('SAVE', () => actions.send({ type: 'save' }));

  // Header shows step indicator during onboarding
  const header = Header({
    title: 'SYNKIO',
    badge: isFirstTime ? `${onboardingStep} of 3` : undefined,
  });

  return PageLayout([header, content, footer]);
}
```

### State Persistence

```typescript
// Storage keys
const STORAGE_KEYS = {
  IS_FIRST_TIME: 'synkio:isFirstTime',
  ONBOARDING_STEP: 'synkio:onboardingStep',
};

// On plugin load
const isFirstTime = await storage.get(STORAGE_KEYS.IS_FIRST_TIME) !== false;
const onboardingStep = await storage.get(STORAGE_KEYS.ONBOARDING_STEP) || 1;

// On onboarding complete
await storage.set(STORAGE_KEYS.IS_FIRST_TIME, false);
```

### Tab Navigation During Onboarding

- Users CAN click other tabs during onboarding (not locked)
- The step indicator stays at current step until they complete it
- "NEXT" button advances the step, not tab clicks
